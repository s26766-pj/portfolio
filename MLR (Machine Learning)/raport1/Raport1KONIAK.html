def plot_confusion_matrix_from_cm(cm, labels=None, title="Confusion Matrix"):
    if labels is None:
        labels = [0, 1]
        
    plt.figure(figsize=(6, 4))
    sns.heatmap(cm, 
                annot=True,
                fmt='d',
                cmap='Blues', 
                cbar=False,
                xticklabels=labels,
                yticklabels=labels)

    plt.xlabel("Predicted label")
    plt.ylabel("True label")
    plt.title(title)
    
    quadrant_labels = {
        (0, 0): "True\nNegative",
        (0, 1): "False\nPositive",
        (1, 0): "False\nNegative",
        (1, 1): "True\nPositive"
    }
    
    ax = plt.gca()
    for (i, j), text_label in quadrant_labels.items():
        ax.text(j + 0.5, i + 0.3, text_label,
                ha='center', va='center',
                color='black', fontsize=10, fontweight='bold')
    
    plt.tight_layout()
    plt.show()

Naiwny bayesowski

S to rne warianty klasyfikatora Naiwnego Bayesa. Wszystkie opieraj si na tej samej podstawowej zasadzie wykorzystania twierdzenia Bayesa przy zaoeniu niezalenoci cech, ale rni si przyjmowanymi zaoeniami dotyczcymi rozkadu danych:

GaussianNB zakada, e cechy maj rozkad normalny, co sprawdza si przy danych cigych.

MultinomialNB najlepiej radzi sobie z danymi reprezentujcymi liczby (np. czstotliwoci sw), czyli danymi dyskretnymi.

BernoulliNB przeznaczony dla danych binarnych, gdzie cechy przyjmuj warto 0 lub 1.

ComplementNB modyfikacja MultinomialNB, ktra lepiej radzi sobie z niezbalansowanymi danymi.

CategoricalNB zaprojektowany specjalnie do pracy z cechami kategorycznymi.

Mimo e s to rne implementacje, wszystkie zaliczamy do rodziny algorytmw Naiwnego Bayesa. Wybr odpowiedniego wariantu zaley od charakterystyki Twoich danych.

Wybraem GaussianNB, poniewa w tym zbiorze danych cechy (np. poziom glukozy, cinienie, BMI itp.) s zmiennymi cigymi, a GaussianNB zakada, e cechy maj rozkad normalny. W wielu przypadkach medycznych parametrw taka aproksymacja jest wystarczajco trafna.

Pozostae warianty Naiwnego Bayesa, takie jak BernoulliNB czy MultinomialNB, s zaprojektowane do pracy z danymi dyskretnymi lub liczbowymi (np. liczbami zdarze, cechami binarnymi), co niekoniecznie odpowiada charakterystyce danych w tym zadaniu. Dlatego GaussianNB jest naturalnym wyborem przy analizie tego typu danych.
